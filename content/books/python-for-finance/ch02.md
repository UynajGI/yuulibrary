对于Python新手来说，Python部署可能看起来并不简单。大量可选安装的库和包也是如此。首先，Python不止一种。Python有多种不同的风格，如CPython、Jython、IronPython和PyPy。还有Python 2.7与3.x世界之间的分歧。¹

即使你决定了版本，部署仍然因其他一些原因而变得困难：

• 解释器（标准CPython安装）仅附带所谓的标准库（例如涵盖典型数学函数）
• 可选的Python包需要单独安装——而且有成百上千个
• 由于依赖关系和特定操作系统的要求，自行编译/构建这些非标准包可能很棘手
• 处理这些依赖关系和随时间保持版本一致性（即维护）通常是繁琐和耗时的
• 某些包的更新或升级可能需要重新编译许多其他包
• 更改或替换一个包可能导致（许多）其他地方的麻烦

幸运的是，有一些可用的工具和策略可以提供帮助。本章涵盖以下有助于Python部署的技术类型：

**包管理器（Package managers）**
像pip和conda这样的包管理器有助于安装、更新和移除Python包；它们还有助于保持不同包的版本一致性。

**虚拟环境管理器（Virtual environment managers）**
像virtualenv或conda这样的虚拟环境管理器允许你并行管理多个Python安装（例如，在单台机器上同时拥有Python 2.7和3.7安装，或测试某个新奇Python包的最新开发版本而无风险）。²

**容器（Containers）**
Docker容器代表完整的文件系统，包含运行特定软件所需的所有系统组件，如代码、运行时或系统工具。例如，你可以在运行macOS或Windows 10的机器上托管的Docker容器中，运行带有Python 3.7安装和相应Python代码的Ubuntu 18.04操作系统。

**云实例（Cloud instances）**
部署用于金融应用的Python代码通常需要高可用性、安全性和性能；这些要求通常只能通过使用专业的计算和存储基础设施来满足，而如今这些基础设施以从相当小到真正大而强大的云实例的形式，以优惠的条件提供。与长期租用的专用服务器相比，云实例的一个好处是用户通常只按实际使用的小时数付费；另一个优势是，如果需要，这类云实例实际上在一两分钟内就可以启动，这有助于敏捷开发和可扩展性。

本章的结构如下：

**conda作为包管理器**
本节介绍conda作为Python包管理器的用法。

**conda作为虚拟环境管理器**
本节重点介绍conda作为虚拟环境管理器的能力。

**使用Docker容器**
本节简要概述Docker作为容器化技术，并重点介绍构建基于Ubuntu的、包含Python 3.7安装的容器。

**使用云实例**
本节展示如何在云端部署Python和Jupyter Notebook——一套强大的、基于浏览器的Python开发工具套件。

本章的目标是在专业基础设施上建立正确的Python安装，配备最重要的工具以及数值、数据分析和可视化包。这样的组合将作为后续章节中实现和部署Python代码的骨干——无论是交互式金融分析代码，还是脚本和模块形式的代码。

## conda作为虚拟环境管理器

根据你选择的安装程序版本，Miniconda提供默认的Python 2.7或3.7安装。conda的虚拟环境管理能力允许你在Python 3.7默认安装的基础上，添加一个完全独立的Python 2.7.x安装。为此，conda提供以下功能：

```txt
创建虚拟环境
conda create --name $ENVIRONMENT_NAME
```

激活环境 conda activate \$ENVIRONMENT\_NAME

停用环境 conda deactivate \$ENVIRONMENT\_NAME

```powershell
移除环境
conda env remove --name $ENVIRONMENT_NAME
```

```txt
导出到环境文件
conda env export > $FILE_NAME
```

```txt
从文件创建环境
conda env create -f $FILE_NAME
```

```txt
列出所有环境
conda info --envs
```

## 使用Docker容器

Docker容器已经席卷了IT世界。虽然这项技术仍然相对年轻，但它已成为高效开发和部署几乎任何类型软件应用的基准之一。

就本书而言，将Docker容器视为一个独立的（"容器化的"）文件系统就足够了，它包含操作系统（如Ubuntu Server 18.04）、（Python）运行时、额外的系统和开发工具以及所需的（Python）库和包。这样的Docker容器可以运行在本地Windows 10机器上，也可以运行在带有Linux操作系统的云实例上。

本节不会深入探讨Docker容器的所有激动人心的细节。它只是简要说明Docker技术在Python部署方面的能力。⁶

## Docker镜像与容器

在继续说明之前，需要区分讨论Docker时的两个基本概念。第一个是Docker镜像（Docker image），可以类比为Python类。第二个是Docker容器（Docker container），可以类比为相应Python类的实例。⁷

在更技术的层面上，Docker词汇表中对镜像有如下定义：

Docker镜像是容器的基础。镜像是一个只读模板，包含创建容器的说明。镜像通常包含堆叠在彼此之上的分层文件系统的联合。镜像没有状态，永远不会改变。

类似地，Docker词汇表中对容器有如下定义，这使得与Python类和类实例的类比更加清晰：

容器是Docker镜像的可运行实例。一个Docker容器包含：一个Docker镜像、一个执行环境以及一组标准指令。

根据操作系统的不同，Docker的安装也有所不同。这就是本节不深入细节的原因。更多信息和相关链接可以在About Docker CE页面上找到。

## 构建Ubuntu和Python的Docker镜像

本节说明如何基于最新版本的Ubuntu构建包含Miniconda以及一些重要Python包的Docker镜像。此外，它还进行一些Linux常规维护，如更新Linux包索引、必要时升级包以及安装某些额外的系统工具。为此，需要两个脚本。一个是bash脚本，负责Linux层面的所有工作。⁸ 另一个是所谓的Dockerfile，用于控制镜像本身的构建过程。

示例2-1中的安装bash脚本由三个主要部分组成。第一部分处理Linux常规维护。第二部分安装Miniconda，而第三部分安装可选的Python包。内联提供了更详细的注释。

## 使用云实例

本节展示如何在DigitalOcean云实例上建立完整的Python基础设施。市面上还有许多其他云提供商，包括领先提供商亚马逊网络服务（AWS）。然而，DigitalOcean以其简洁性和相对低廉的小型云实例（称为Droplet）费率而闻名。最小的Droplet（通常足以满足探索和开发目的）每月仅需5美元，或每小时0.007美元。使用按小时计费，因此你可以轻松启动一个Droplet使用2小时，然后销毁它，仅需支付0.014美元。⁹

本节的目标是在DigitalOcean上建立一个Droplet，配备Python 3.7安装及通常需要的包（如NumPy、pandas），并配合密码保护和安全套接层（SSL）加密的Jupyter Notebook服务器安装。该服务器安装将提供三种可通过常规浏览器使用的主要工具：

**Jupyter Notebook**
一个流行的交互式开发环境，内置多种语言内核（如Python、R和Julia）。

**终端**
可通过浏览器访问的系统shell实现，允许所有典型的系统管理任务，并使用Vim和git等有用工具。

**编辑器**
基于浏览器的文件编辑器，支持多种编程语言和文件类型的语法高亮，以及典型的文本/代码编辑能力。

在Droplet上安装Jupyter Notebook后，你可以通过浏览器进行Python开发和部署，无需通过安全外壳（SSH）访问登录云实例。

为实现本节的目标，需要以下文件：

**服务器设置脚本** 该脚本协调所有必要的步骤，例如将其他文件复制到Droplet并在其上运行。

**Python和Jupyter安装脚本** 安装Python、附加包和Jupyter Notebook，并启动Jupyter Notebook服务器。

**Jupyter Notebook配置文件** 用于配置Jupyter Notebook服务器，如密码保护等。

以下小节将反向逐一介绍这些文件。

## RSA公钥和私钥

为了通过任意浏览器创建到Jupyter Notebook服务器的安全连接，需要由RSA公钥和私钥组成的SSL证书。通常，这样的证书应来自所谓的证书颁发机构（CA）。然而就本书而言，自生成的证书"足够好"。¹⁰ 生成RSA密钥对的一个流行工具是OpenSSL。以下简短交互式会话展示了如何生成适用于Jupyter Notebook服务器的证书（根据提示输入你自己的国家名称和其他字段值）：

```shell
~/cloud$ openssl req -x509 -nodes -days 365 -newkey \
> rsa:1024 -out cert.pem -keyout cert.key
Generating a 1024 bit RSA private key
..++++++
......++++++
writing new private key to 'cert.key'
```

You are about to be asked to enter information that will be incorporated into your certificate request.
What you are about to enter is what is called a Distinguished Name or a DN.
There are quite a few fields but you can leave some blank
For some fields there will be a default value,
If you enter '.', the field will be left blank.

```txt
Country Name (2 letter code) [AU]:DE
State or Province Name (full name) [Some-State]:Saarland
Locality Name (eg, city) []:Voelklingen
```

```txt
Organization Name (eg, company) [Internet Widgits Pty Ltd]:TPQ GmbH
Organizational Unit Name (eg, section) []:Python for Finance
Common Name (e.g. server FQDN or YOUR name) []:Jupyter
Email Address []:team@tpq.io
~/cloud$ ls
cert.key    cert.pem
~/cloud$
```

两个文件 `cert.key` 和 `cert.pem` 需要复制到Droplet，并在Jupyter Notebook配置文件中引用。接下来介绍该文件。

## Jupyter Notebook配置文件

正如文档中所解释的，公共Jupyter Notebook服务器可以安全部署。Jupyter Notebook的功能之一是密码保护。为此，`notebook.auth` 子包中有一个名为 `passwd()` 的密码哈希码生成函数。以下代码生成了一个密码哈希码，其中 `jupyter` 为密码本身：

```txt
~/cloud$ ipython
Python 3.7.0 (default, Jun 282018, 13:15:42)
Type 'copyright', 'credits' or 'license' for more information
IPython 6.5.0 -- An enhanced Interactive Python. Type '?' for help.
```

In [1]: from notebook.auth import passwd

```txt
In [2]: passwd('jupyter')
Out[2]: 'sha1:d4d34232ac3a:55ea0ffd78cc3299e3e5e6ecc0d36be0935d424b'
```

```txt
In [3]: exit
```

此哈希码需要放置在示例2-3所示的Jupyter Notebook配置文件中。该配置文件假定RSA密钥文件已复制到Droplet的 `/root/.jupyter/` 文件夹。

示例2-3. Jupyter Notebook配置文件

```shell

### Jupyter Notebook配置文件

SSL加密
将以下文件名（以及使用的文件）替换为你自己的选择/文件

c. NotebookApp.keyfile = u'/root/.jupyter/cert.key'

IP地址和端口
设置ip为'*'以绑定云实例的所有IP地址
```

```python

建议设置一个已知的、固定的默认端口用于服务器访问

密码保护
此处：密码为'jupyter'
将哈希码替换为你强密码对应的哈希码

不启动浏览器选项
防止Jupyter尝试打开浏览器
c. NotebookApp.open_browser = False
```

## 组织Droplet设置的脚本

第二个bash脚本（用于设置Droplet）是最短的（示例2-5）。它主要将所有其他文件复制到Droplet（其IP地址作为参数传入）。最后一行启动 `install.sh` bash脚本，该脚本执行安装本身并启动Jupyter Notebook服务器。

示例2-5. 设置Droplet的bash脚本

```shell
#!/bin/bash
#
### 设置DigitalOcean Droplet
### 基础Python栈和Jupyter Notebook
#


#
从参数获取IP地址
MASTER_IP=$1

复制文件
scp install.sh root@$MASTER_IP):
scp cert.* jupyter_notebook_config.py root@$MASTER_IP):

执行安装脚本
ssh root@$MASTER_IP} bash /root/install.sh
```

现在一切都就绪，可以尝试设置代码。在DigitalOcean上，使用类似以下选项创建一个新的Droplet：

操作系统 Ubuntu 18.10 x64（撰写本文时可用的最新版本）
规格 1核，1 GB，25 GB SSD（最小的Droplet）
数据中心区域 法兰克福（因为作者居住在德国）
SSH密钥 添加（新的）SSH密钥用于免密码登录 ¹¹
Droplet名称 可以使用预设名称，或选择如py4fi之类的名称

点击Create按钮将启动Droplet创建过程，通常需要约一分钟。设置过程的主要成果是IP地址，例如，如果你选择法兰克福作为数据中心位置，IP地址可能是46.101.156.199。现在设置Droplet非常简单：

整个过程可能需要几分钟。当Jupyter Notebook服务器显示类似以下消息时，表示完成：

在任何现代浏览器中，访问以下地址即可连接到正在运行的Jupyter Notebook服务器（注意https协议）：

https://46.101.156.199:8888

在可能要求你添加安全例外后，Jupyter Notebook登录屏幕将提示输入密码（此处为jupyter）。现在你就可以通过Jupyter Notebook在浏览器中开始Python开发，通过终端窗口使用IPython，或使用文本文件编辑器。其他文件管理功能（如文件上传、文件删除和文件夹创建）也同样可用。

## 结论

Python是首选编程语言和技术平台——不仅对本书而言如此，对几乎所有领先金融机构也是如此。然而，Python部署充其量可能很棘手，有时甚至令人厌倦和紧张。幸运的是，近年来出现了几种有助于解决部署问题的技术。开源conda在Python包和虚拟环境管理两方面都有帮助。Docker容器则更进一步，可以在技术上屏蔽的"沙箱"（即容器）中轻松创建完整的文件系统和运行时环境。再进一步，DigitalOcean等云提供商在专业管理和保护的数据中心内，在几分钟内提供按小时计费的计算和存储容量。这与Python 3.7安装和安全Jupyter Notebook服务器安装的结合，为Python开发和在Python-for-finance项目背景下的部署提供了专业环境。

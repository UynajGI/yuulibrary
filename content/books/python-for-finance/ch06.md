# 面向对象编程

软件工程的目标是控制复杂性，而不是制造复杂性。

—Pamela Zave

面向对象编程（Object-Oriented Programming, OOP）是当今最流行的编程范式之一。如果使用得当，它与过程式编程（Procedural Programming）相比具有许多优势。在许多情况下，OOP 似乎特别适合金融建模和实现金融算法。然而，也有许多批评者对其中的某些方面甚至整个范式持怀疑态度。本章采取中立立场，将 OOP 视为一种重要工具，它可能不是解决每个问题的最佳方法，但应该是从事金融工作的程序员和宽客（Quants）应该掌握的工具。

使用 OOP 会带来一些新的语言术语。就本书和本章而言，最重要的术语如下（更多内容将在后面介绍）：

## 类（Class）

某种类型对象的抽象定义。例如，人类。

对象（Object）

类的一个实例。例如，Sandra。

## 属性（Attribute）

类的一个特征（类属性）或类实例的特征（实例属性）。例如，属于哺乳动物、男性或女性，或眼睛的颜色。

## 方法（Method）

类或其实例可以实现的操作。例如，行走。

## 参数（Parameter）

方法接受的输入，用于影响其行为。例如，三步。

实例化（Instantiation）

基于抽象类创建特定对象的过程。

转换成 Python 代码，实现人类示例的简单类可能如下所示：

支持使用 OOP 的若干人为因素：

## 自然思维方式

人类思维通常围绕现实世界或抽象对象展开，如汽车或金融工具。OOP 适合用其特征来建模此类对象。

## 降低复杂性

通过不同的方法，OOP 有助于降低问题或算法的复杂性，并逐步建立模型。

## 更友好的用户界面

OOP 在许多情况下允许更好的用户界面和更紧凑的代码。例如，在查看 NumPy 的 ndarray 类或 pandas 的 DataFrame 类时，这一点变得很明显。

## Python式的建模方式

不论 OOP 的利弊如何，它只是 Python 中的主导范式。这就是"Python 中一切都是对象"这一说法的来源。OOP 还允许程序员构建自定义类，其实例的行为与标准 Python 类的每个实例一样。

支持使用 OOP 的若干技术因素：

## 抽象（Abstraction）

使用属性和方法可以构建抽象、灵活的对象模型，关注相关的内容，忽略不需要的内容。在金融领域，这可能意味着有一个通用类来以抽象方式建模金融工具。这样的类的实例将是具体的金融产品，例如由投资银行设计提供的。

## 模块化（Modularity）

OOP 简化了将代码分解为多个模块的过程，这些模块随后被链接在一起形成完整的代码库。例如，对股票上的欧式期权进行建模可以通过一个类实现，也可以通过两个类来实现，一个用于标的股票，一个用于期权本身。

## 继承（Inheritance）

继承是指一个类可以从另一个类继承属性和方法的概念。在金融领域，从一般的金融工具开始，下一层可以是一般的衍生工具，然后是欧式期权，然后是欧式看涨期权。每个类可能继承自更高层级类的属性和方法。

## 聚合（Aggregation）

聚合是指一个对象至少部分由多个可能独立存在的其他对象组成的情况。建模欧式看涨期权的类可能将标的股票和相关贴现短期利率的其他对象作为属性。表示股票和短期利率的对象也可以独立地被其他对象使用。

## 组合（Composition）

组合类似于聚合，但这里的单个对象不能彼此独立存在。考虑一个定制化的利率互换，包含固定端和浮动端。这两端不能独立于互换本身存在。

## 多态（Polymorphism）

多态可以有多种形式。在 Python 语境中，特别重要的是所谓的鸭子类型（Duck Typing）。这指的是标准操作可以在许多不同的类及其实例上实现，而不必确切知道处理的是什么对象。对于金融工具类，这可能意味着可以调用 get_current_price() 方法，而与对象的具体类型（股票、期权、互换）无关。

## 封装（Encapsulation）

这个概念指的是使类中的数据只能通过公共方法访问的方法。建模股票的类可能有一个属性 current_stock_price。封装将通过方法 get_current_stock_price() 提供对属性值的访问，并将数据对用户隐藏（即使其私有）。这种方法可能避免简单地操作和可能更改属性值带来的意外影响。然而，在 Python 类中使数据私有的方式是有局限的。

在更高层面上，这些方面中的许多可以归纳为软件工程的两个总体目标：

## 可复用性（Reusability）

继承和多态等概念提高了代码的可复用性，提高了程序员的效率和生产能力。它们还简化了代码维护。

## 无冗余性（Nonredundancy）

同时，这些方法允许构建几乎无冗余的代码，避免重复的实现工作，并减少调试和测试工作以及维护工作量。它们还可能带来更小的整体代码库。

## Python数据模型

前一节的示例突出了所谓的 Python 数据模型（Data Model）或对象模型的一些方面。Python 数据模型允许设计出与 Python 基本语言结构一致交互的类。除其他外，它支持（参见 Ramalho (2015)，第4页）以下任务和结构：

• 迭代（Iteration）

• 集合处理（Collection Handling）

• 属性访问（Attribute Access）

• 运算符重载（Operator Overloading）

• 函数和方法调用（Function and Method Invocation）

• 对象的创建和销毁（Object Creation and Destruction）

• 字符串表示（例如用于打印）

• 管理上下文（Managed Contexts，即 with 块）

## 属性值全为零的新 Vector 对象。

如前多次所示，+ 和 \* 运算符可以应用于几乎任何 Python 对象。其行为通过特殊方法 \_\_add\_\_ 和 \_\_mul\_\_ 定义：

```python
In [93]: class Vector(Vector):
    def__add__(self, other):
    x = self.x + other.x
    y = self.y + other.y
```

## 结语

本章从理论和 Python 示例两个角度介绍了面向对象编程的概念和方法。OOP 是 Python 中使用的主要编程范式之一。它不仅允许建模和实现相当复杂的应用程序，而且还允许创建自定义对象，由于灵活的 Python 数据模型，这些对象的行为类似于标准 Python 对象。尽管有许多批评者反对 OOP，但可以肯定地说，它为 Python 程序员和宽客提供了强大的工具，在达到一定复杂性时非常有帮助。第五部分开发和讨论的衍生品定价包就是这样一个案例，其中 OOP 似乎是处理固有复杂性和抽象要求的唯一明智的编程范式。

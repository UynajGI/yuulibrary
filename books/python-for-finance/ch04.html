    数据数组#

    前一章展示了Python提供了一些相当有用和灵活的通用数据结构。特别是，列表对象可以被视为具有许多便利特性和应用领域的真正主力。使用这种灵活（可变）的数据结构是有代价的，表现为相对较高的内存使用、较慢的性能，或两者兼有。然而，科学和金融应用通常需要在特殊数据结构上进行高性能操作。在这方面最重要的数据结构之一就是数组（Array）。数组通常将相同数据类型的其他（基本）对象按行和列组织起来。

    暂时假设只有数字是相关的，尽管这个概念也可以推广到其他类型的数据。在最简单的情况下，一维数组在数学上表示一个向量，通常由浮点数对象在内部表示的真实数组成。它仅由单行或单列元素组成。在更常见的情况下，数组表示一个 i × j 的元素矩阵。这个概念可以推广到三维中的 i × j × k 立方体元素，以及一般的 n 维数组，其形状为 $i \times j \times k \times l \times$

    线性代数和向量空间理论等数学学科表明，这种数学结构在许多科学学科和领域中具有高度重要性。因此，拥有一个专门设计用于方便高效处理数组的数据结构类是富有成效的。这正是Python库NumPy及其强大的 ndarray 类的用武之地。在下一节介绍这个类之前，本节将说明处理数组的两种替代方案。

    常规NumPy数组#

    用列表对象组合数组结构是可行的，但并不是很方便，而且 list 类的设计初衷也不是为了实现这一目标。它的范围更广泛、更通用。array 类稍专门化一些，提供了一些处理数据数组的有用特性。然而，一个真正专门化的类对于处理数组类型的结构将非常有益。

    内存布局#

    当使用 np.zeros() 初始化 ndarray 对象时（如"多维" on page 94 所示），提供了一个可选参数用于内存布局。这个参数大致指定了数组中的哪些元素在内存中相邻存储（连续）。处理小数组时，这对数组操作的性能几乎没有什么可衡量的影响。然而，当数组变大，并且取决于要实现的（金融）算法时，情况可能会不同。这时内存布局就变得重要了（参见例如 Eli Bendersky 的文章 "Memory Layout of Multi-Dimensional Arrays"）。

    我们可以将性能结果总结如下：

    • 计算所有元素的和时，内存布局并不重要。

    • C 顺序的 ndarray 对象在行和列上的求和都更快（绝对速度优势）。

    • 对于 C 顺序（行优先）的 ndarray 对象，按行求和相对比按列求和更快。

    • 对于 F 顺序（列优先）的 ndarray 对象，按列求和相对比按行求和更快。

    结语#

    NumPy 是 Python 中进行数值计算的首选包。ndarray 类专门设计用于方便高效地处理（大型）数值数据。强大的方法和 NumPy 通用函数允许编写向量化代码，从而避免在 Python 层面上使用缓慢的循环。本章介绍的许多方法也适用于 pandas 及其 DataFrame 类（参见第5章）。

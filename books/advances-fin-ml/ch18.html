    熵特征#

    18.1 动机#

    价格序列传达了供需力量的信息。在完美市场中，价格是不可预测的，因为每个观测传达了关于产品或服务的所有已知信息。当市场不完美时，价格以部分信息形成，由于某些代理人比其他人了解更多，他们可以利用这种信息不对称。估计价格序列的信息含量，并形成 ML 算法可以学习可能结果的特征将是有帮助的。例如，ML 算法可能发现当价格携带少量信息时动量下注更有利可图，而当价格携带大量信息时均值回归下注更有利可图。在本章中，我们将探索确定价格序列中包含的信息量的方法。

    18.5 编码方案#

    估计熵需要对消息进行编码。在本节中，我们将回顾文献中使用的一些基于收益的编码方案。虽然以下未讨论，但建议从分数阶（而非整数）差分序列（第 4 章）编码信息，因为它们仍然包含一些记忆。

    18.5.1 二进制编码#

    熵率估计需要连续变量的离散化，以便每个值可以从有限字母表中分配一个代码。例如，收益流 rt 可以根据符号编码，rt > 0 为 1，rt < 0 为 0，去除 rt = 0 的情况。当收益序列从价格条（即包含在两个对称水平障碍之间波动的价格的条，以起始价格为中心）采样时，二进制编码自然出现，因为 |rt| 近似恒定。

    当 |rt| 可以采用广泛的结果范围时，二进制编码丢弃了潜在有用的信息。当使用受 tick 数据非均匀性质导致的异方差性影响的日内时间条时尤其如此。部分解决这种异方差性的一种方法是按从属随机过程采样价格。这方面的例子是交易条和成交量条，它们包含固定数量的交易或固定成交量的交易（见第 2 章）。通过在这种非时间顺序的、市场驱动的时钟中操作，我们在高活动期间更频繁地采样，在低活动期间更少采样，从而规范化 |rt| 的分布并减少对大字母表的需求。

    18.5.2 分位数编码#

    除非使用价格条，否则可能需要两个以上的代码。一种方法是根据 rt 所属的分位数分配代码。分位数边界使用样本内期间（训练集）确定。整个样本内每个字母将分配相同数量的观测，样本外每个字母的观测数接近相同。使用该方法时，某些代码跨越 rt 范围的更大比例。这种均匀（样本内）或接近均匀（样本外）的代码分布往往平均增加熵读数。

    18.8 熵的几个金融应用#

    在本节中，我们将介绍熵在金融市场建模中的一些应用。

    18.8.1 市场效率#

    当套利机制利用了完整的机会集时，价格瞬时反映全部可用信息，变得不可预测（即鞅），没有可辨别的模式。相反，当套利不完美时，价格包含不完整的信息量，从而产生可预测的模式。当字符串包含冗余信息时出现模式，这使其能够压缩。字符串的熵率确定其最优压缩率。熵越高，冗余越低，信息内容越大。因此，价格字符串的熵告诉我们在给定时间点的市场效率程度。「解压的」市场是有效市场，因为价格信息是非冗余的。「压缩的」市场是无效市场，因为价格信息是冗余的。泡沫在压缩（低熵）市场中形成。

    18.8.2 最大熵生成#

    在一系列论文中，Fiedor [2014a, 2014b, 2014c] 提出使用 Kontoyiannis [1997] 估计价格序列中存在的熵量。他认为，在可能的未来结果中，最大化熵的结果可能是最有利可图的，因为它是最不被频率统计模型预测到的。它是最有可能触发止损的黑天鹅场景，从而产生将加强和加剧运动的反馈机制，导致收益时间序列符号的运行。

    在大小为 V 的成交量条内，我们可以根据某种算法（如 tick 规则或 Lee-Ready 算法）将 tick 分类为买入或卖出。令 V^B^τ 为成交量条 τ 中买入 tick 的成交量之和，V^S^τ 为成交量条 τ 中卖出 tick 的成交量之和。Easley 等 [2012a, 2012b] 注意到 E[|V^B^τ − V^S^τ|] ≈ αμ 且预期总成交量为 E[V^B^τ + V^S^τ] = αμ + 2ϵ。通过使用成交量时钟（Easley 等 [2012c]），我们可以外生地设定 E[V^B^τ + V^S^τ] = αμ + 2ϵ = V 的值。这意味着在成交量时钟下，PIN 简化为

    其中 。注意 2v^B^τ − 1 代表订单流不平衡 OIτ，它是一个有界实值变量，OIτ ∈ [−1, 1]。因此，VPIN 理论提供了知情交易概率（PIN）与成交量时钟下订单流不平衡持续性之间的正式联系。该微观结构理论的更多细节见第 19 章。

    持续的订单流不平衡是逆向选择的必要非充分条件。做市商要向知情交易者提供流动性，该订单流不平衡 |OIτ| 也必须是相对不可预测的。换言之，当做市商对订单流不平衡的预测准确时，即使 |OIτ| ≫ 0，他们也不会被逆向选择。为了确定逆向选择的概率，我们必须确定订单流不平衡有多不可预测。我们可以通过应用信息论来确定这一点。

    考虑一个长的符号序列。当该序列包含很少的冗余模式时，它包含一定程度的复杂性，使其难以描述和预测。Kolmogorov [1965] 形式化了冗余与复杂性之间的联系。在信息论中，无损压缩是用尽可能少的比特完美描述序列的任务。序列包含的冗余越多，可以实现的压缩率越高。熵表征了源的冗余，因此表征了其 Kolmogorov 复杂性和可预测性。我们可以利用序列冗余与其不可预测性（做市商）之间的联系来推导逆向选择的概率。

    这里我们将讨论一种特定程序，该程序将逆向选择概率推导为订单流不平衡中根深蒂固的复杂性的函数。第一，给定以 τ = 1, ..., N 索引的成交量条序列，每条大小为 V，我们确定分类为买入的成交量部分 v^B^τ ∈ [0, 1]。第二，我们计算 {v^B^τ} 上的 q 分位数，定义一组 K 的 q 个不相交子集，K = {K1, ..., Kq}。第三，我们产生从每个 v^B^τ 到不相交子集之一的映射 f: v^B^τ → {1, ..., q}，其中 f[v^B^τ] = i ⇔ v^τB ∈ Ki，∀i ∈ [1, q]。第四，我们通过将每个值 v^B^τ 分配给它所属子集 K 的索引 f[v^B^τ] 来量化 {v^B^τ}。这将订单不平衡集 {v^B^τ} 转换为量化消息 X = {f[v^B^1], f[v^B^2], ..., f[v^B^N]}。第五，我们使用 Kontoyiannis 的 Lempel-Ziv 算法估计熵 H[X]。第六，我们推导累积分布函数 F[H[X]]，并使用 {F[H[Xτ]]}τ=1,...,N 的时间序列作为预测逆向选择的特征。

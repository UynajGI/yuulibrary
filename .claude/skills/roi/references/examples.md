# ROI 输出示例

## 示例 1：有明确技术债的通用项目

状态摘要：TypeScript 项目，lint 报告 34 warnings（23 个 unused imports + 11 个 any 类型），测试通过，构建通过。最近 3 个 commit 为新功能开发。

```
## ROI 建议

> 34 lint warnings，全部是 unused imports（23）和 any 类型（11）。无阻断性问题。

### 🥇 优先级 1：配置 ESLint auto-fix 处理 unused imports
- **为什么**：23 个 unused imports 是纯机械问题，`eslint --fix` 一键解决。不需要人工逐个删。
- **影响**：I=4（23 处，所有文件）/ E=短（配置 + 运行 <5 分钟）
- **预估**：5 分钟

### 🥈 优先级 2：给 CI 加 lint check，block 新增 any 类型
- **为什么**：11 个 any 类型是历史债，但如果不阻止新增，永远还不完。加 CI rule 后至少不会恶化。
- **影响**：I=5（永久复利，每次 PR 受益）/ E=中
- **预估**：20 分钟

### 🥉 优先级 3：更新 CLAUDE.md 技术栈说明
- **为什么**：新依赖加了 3 个但没有更新 CLAUDE.md，下次 Agent 不知道用什么库。
- **影响**：I=3（跨会话复用）/ E=短
- **预估**：5 分钟

### 其他可考虑
- 逐个修 11 个 any 类型（需要理解业务逻辑，投入大，不如先 block 新增）
```

## 示例 2：干净项目——没有明显 Bug

状态摘要：Python web 项目，lint 0 issues，测试全部通过（142 tests），CI green。最近 commit 为文档更新。

```
## ROI 建议

> 0 issues，142 tests 全过，CI green。当前状态干净。

### 🥇 优先级 1：加集成测试覆盖核心 API
- **为什么**：142 个测试全是单元测试，0 个集成测试。API 层的回归靠人工——这是最大的质量盲区。
- **影响**：I=4（保护最关键的 API 路径）/ E=中
- **预估**：1 小时（3-5 个核心 endpoint）

### 🥈 优先级 2：CLAUDE.md 补充部署流程
- **为什么**：当前 CLAUDE.md 只有开发命令，没有部署/回滚步骤。换 Agent 或同事接手会踩坑。
- **影响**：I=3（跨会话/跨人复用）/ E=短
- **预估**：15 分钟

### 其他可考虑
- 升级依赖（非紧急，无安全漏洞，排在最后）
```

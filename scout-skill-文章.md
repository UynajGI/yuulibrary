# 我做了一个 AI 会话开局顾问，叫 scout

卡神（数字生命卡兹克）之前开源了洁癖（neat-freak），一个会话收尾的 skill，跑一下 `/neat`，自动帮你把项目文档、CLAUDE.md、记忆全部审查一遍，该改的改，该删的删，然后给你一份变更摘要。

我装上之后用了快一个月，确实好用，每次任务做完不跑一遍 `/neat` 就浑身难受。

但用着用着，我发现一个问题。

洁癖解决的是「收尾」——做完事之后把学到的东西沉淀下来。但没人解决「开局」——打开项目不知道干什么的时候，怎么办。

我自己就经常这样。打开终端，看着几百个文件的项目，git status 干净，CI 绿着，没有什么紧急的 Bug。但可做的事一大堆——加测试、改 lint 规则、重构一个模块、更新文档、加新功能……先做哪个？

每次都是凭感觉。感觉哪个急做哪个，感觉哪个简单先做哪个。然后经常做了一两个小时，发现方向不对，前面的时间全浪费了。

我就想，能不能做一个和洁癖配对的东西——洁癖是存档，这个是读档。打开项目跑一下，它告诉我现在做什么最划算。

于是就有了 `scout` 的第一版。逻辑很朴素：扫一遍项目状态（git、lint、测试、记忆），把能做的事全列出来，然后按优先级排序，输出三条建议。

第一个版本其实就能用了，但排序逻辑很糙。基本上就是「紧急的排前面，简单的排前面」，加点「修流程比修 Bug 划算」的常识。能用，但不够好。

然后正巧前阵子，我在知乎上刷到了一篇文章。

文章讲的是 Jacob Steinhardt 在 Stanford 读博时写的一篇博客，叫 **《Research as a Stochastic Decision Process》**（把研究当作随机决策过程）。

里面有一个观点，我读完直接拍大腿。

Steinhardt 说，大多数人面对一堆不确定的任务时，会默认「先做简单的」。但这个直觉是有害的。因为简单的做完了，碰到真正的难点才发现走不通，前面的功夫全白搭了。

正确的做法是**按信息率排序**。他给了一个公式：

**λ = log(1/p) / T**

p 是你觉得这事能做通的概率，T 是大概要花多少时间。λ 就是信息率——你每花一分钟，能获得多少「这事到底行不行」的信息。

什么意思呢？举个例子。两个任务：
- A：半小时能做完，但你只有 40% 的把握
- B：要搞四个小时，但你有 90% 的把握

你的直觉肯定是先做 B，毕竟它稳嘛。但 A 的 λ 是 0.23，B 的 λ 是 0.009。差了几十倍。应该先做 A——它虽然悬，但半小时就能给你一个明确的答案。行就行，不行就换路，不心疼。

这个逻辑放到项目里特别好用。比如你 lint 挂了三十几个 warning，跑一下 `eslint --fix` 三十秒的事——但它马上告诉你，这三十几个 warning 里哪些是纯机械的（一键修掉），哪些是真正需要改逻辑的。你就知道后面要花多少时间了。

反过来，那些「重构整个模块」的计划，成功率很高，但信息率很低——你基本能确定它能做成，早做晚做差别不大。

我当时看完就想，这不就是我缺的东西吗？

`scout` 第一版的排序问题，就是因为它只考虑了「紧急程度」和「难度」，没有考虑「信息获取的效率」。Steinhardt 这个框架正好补上了。

然后我就把文章里几个核心概念全融进了 `scout`：

一个是**降风险模式 vs 执行模式**。很多时候你以为自己在做事，其实只是在推迟判断。比如你花四个小时实现了一个方案，做完才发现方向错了。这四小时真正的问题不是你做得慢，而是你一开始就该花十分钟做个最小可行版本，先验证方向对不对。`scout` 现在排序的时候，降风险型的动作自动排前面。

一个是**失败也是信息，但前提是你问了为什么**。一次实现失败了，不要只记「这方法不行」，要问「这类方法为什么不行」。这样一次失败能帮你排除一整类方案，而不是一个具体实现。`scout` 在识别候选动作的时候，会检查项目记忆里有没有类似的失败记录，避免重复踩坑。

现在还加了一个降风险技巧的参考文件，里面有上限测试、基线测试、最小可行测试这些东西，需要的时候可以翻。

---

现在 `scout` 做四件事：

1. 侦查项目状态——git、lint、测试、CLAUDE.md、记忆，全跑一遍，给你具体数字
2. 把能做的事全列出来，合并同类项
3. 按信息率排序——降风险的优先，λ 高的优先，复利型的优先
4. 输出三条建议，每条告诉你为什么、要多久

长这样：

```
## ROI 建议

> 0 issues，142 tests 全过，CI green。当前状态干净。

### 🥇 优先级 1：加集成测试覆盖核心 API
- **为什么**：142 个测试全是单元测试，0 个集成测试。API 层的回归靠人工。
- **模式**：降风险
- **预估**：1 小时

### 🥈 优先级 2：CLAUDE.md 补充部署流程
- **为什么**：当前只有开发命令，没有部署/回滚步骤。
- **预估**：15 分钟
```

就这么个东西。每次打开项目不知道该干嘛了，就 `/scout` 一下。

和洁癖配合起来就是：
- 开局 `/scout` → 该干嘛
- 收尾 `/neat` → 存档，把学到的东西写进去

两个 skill 共用一个理念：**把经验系统化，下一次更好。**
